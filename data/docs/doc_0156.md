# Technical Design: Critical Care Network Segmentation and EHR Gateway Upgrade

## Executive Summary
This document outlines the proposed architecture to modernize the clinical network backbone at Riverside General Hospital. Following the Q3 2024 audit findings, the IT Infrastructure team—led by Senior Architect Dr. Elena Vasquez—will implement micro-segmentation for the Epic EHR environment. The goal is to reduce lateral movement risk while maintaining sub-50ms latency to bedside terminals in the East Wing ICU.

## Current State and Risk Assessment
The legacy flat VLAN design, deployed in March 2019, currently supports 1,240 clinical endpoints across four nursing towers. Recent penetration testing by RedShield Consulting revealed uncontrolled east-west traffic between the radiology PACS subnet and the pharmacy inventory VLAN. Additionally, the existing Cisco Nexus 9306 core switches are approaching end-of-support on 14 October 2025, creating urgency for replacement.

Key concerns include:
- Unencrypted HL7 v2.x messaging traversing shared Layer 2 domains
- Single points of failure in the North Data Center aggregation layer
- Inability to enforce device-level Zero Trust policies for 340 portable infusion pumps

## Proposed Architecture
We will deploy a three-tier spine-leaf fabric using Arista 7280R3 switches, with dedicated clinical zones. The Epic Hyperspace gateway cluster will migrate to redundant virtual appliances running on VMware vSphere 8.0 Update 3, hosted in the primary and secondary data centers.

Implementation phases:
- Phase 1 (1 November 2024 – 15 December 2024): Spine-leaf cabling and switch provisioning
- Phase 2 (6 January 2025 – 28 February 2025): EHR gateway cutover and VLAN migration
- Phase 3 (10 March 2025): Decommission legacy Nexus 9356 pairs and archive configurations

Network access control will be enforced via Cisco ISE 3.2, profiling every endpoint against the CMDB maintained by James Whitfield’s asset management team.

## Security and Compliance Controls
All clinical traffic will be encrypted using MACsec on inter-switch links and TLS 1.3 for API communications with the Epic Chronicles database. Access logs will feed into the existing Splunk Enterprise Security instance, retained for seven years per HIPAA retention mandates. Dr. Vasquez will conduct weekly architecture review meetings every Tuesday at 09:00 EST until project closure on 31 March 2025.

## Resource Requirements and Timeline
The capital budget requests $487,500 for hardware, licensing, and professional services. Staffing requires two network engineers at 50% FTE for 20 weeks, plus one Epic-certified integration specialist contracted through Apex Health IT. All maintenance windows are restricted to Sundays 02:00–06:00 EST to avoid disruption to the 24-hour emergency department and surgical suites.

Approval is requested by 25 October 2024 to maintain the procurement schedule.
