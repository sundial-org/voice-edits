# Meridian Commerce Platform

Meridian is a modern headless e-commerce engine designed for high-velocity retail operations. Built to serve merchants processing over 2.4 million orders per month, the platform separates storefront presentation from core commerce logic, enabling teams to deploy custom experiences without touching backend inventory systems.

## Architecture Overview

The system runs on a multi-tenant Kubernetes cluster with regional edge caches in Frankfurt, Singapore, and Virginia. At its core, Meridian uses an event-sourced transaction model that guarantees consistency across payment, fulfillment, and analytics pipelines. All public APIs conform to the Open Commerce Specification v3.1 released in March 2024.

Key design principles include:
- Decoupled storefronts via GraphQL and REST gateways
- Real-time inventory synchronization using Apache Kafka streams
- Automated rollback protocols with a 45-second recovery window
- PCI-DSS Level 1 compliance enforced by default

## Core Capabilities

Meridian ships with a comprehensive feature set tailored to mid-market and enterprise sellers:

- Dynamic pricing engine supporting 12 concurrent promotion rules per SKU
- Multi-currency checkout with localized tax calculation for 38 countries
- AI-driven fraud detection trained on 18 months of transaction history
- Headless CMS integration with native support for Contentful and Sanity
- Warehouse management connectors for ShipBob, Deliverr, and in-house WMS tools

The admin dashboard provides role-based access controls, allowing marketing, logistics, and finance teams to operate within isolated workspaces without cross-contamination of sensitive order data.

## Installation and Setup

Clone the repository and install dependencies using your preferred package manager. Configure environment variables for database connections, Redis caching, and the external payment provider. A Docker Compose stack is included for local development; production deployments should use the Helm charts maintained in the `/deploy` directory.

Minimum system requirements include 8 GB RAM and 4 vCPUs per node. The platform was originally developed by Elena Vasquez and the engineering team at Northbridge Labs in 2019, and has since evolved into an open-core project governed by the Meridian Foundation.

## Deployment Notes

Production releases follow a blue-green strategy with automated canary analysis. The CI pipeline runs 340 unit tests and 56 integration tests before any artifact reaches the staging environment. Monitoring relies on Prometheus and Grafana dashboards pre-configured with alerting thresholds for latency, error rate, and inventory drift.

For support, consult the documentation portal or open an issue in the public tracker. Enterprise customers may contact the solutions engineering team directly for architecture reviews and migration planning from legacy platforms such as Magento 2 or Shopify Plus.
