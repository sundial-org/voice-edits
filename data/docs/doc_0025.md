# Resonance Collective — Distributed Audio Pipeline Design Document

## Overview
Resonance Collective operates three studios across Berlin, Lisbon, and Tokyo. This document defines the technical architecture for our 2025 shared-production environment, codenamed Project Aurora, enabling 48 kHz multitrack collaboration with sub-12 ms latency.

## System Architecture
The backbone runs on redundant 10 Gbps fiber links between nodes. Each site hosts identical reference monitors—Genelec 8351B—calibrated to 83 dB SPL at the mix position. We use a custom Dante/AES67 bridge developed by engineer Lin Zhou, deployed first on March 14, 2024.

## Collaboration Protocol
To preserve sonic intent across time zones, we enforce a strict session format:
- All stems exported as 32-bit float WAV at 48 kHz / 24-bit
- Metadata embedded using BWAV with iXML v3.0 descriptors
- Revisions tracked in Perforce with changelists named after the track ISRC
- Daily sync windows: 09:00 UTC and 21:00 UTC

## Security and Access
Access to the master vault requires hardware-based FIDO2 keys assigned to each of the 14 collective members. Two-factor authentication is mandatory for remote console access.

## Milestones
Phase one—network validation—completes by September 30, 2025. Phase two—live remote mixing—launches January 12, 2026. The collective targets a public release of the Aurora toolkit by June 2026.
