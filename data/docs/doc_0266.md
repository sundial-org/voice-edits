# TerraGrid PRD: Real-Time Carbon Intensity API v2.0

## Overview
TerraGrid is building the operating system for grid decarbonization. This document defines the v2.0 release of our Real-Time Carbon Intensity API, which delivers marginal emissions factors to enterprise clients every 15 minutes. Led by Product Manager Elena Vasquez, the initiative targets a Q3 2025 launch to support new EU CBAM reporting rules and replace our legacy v1 endpoints by October 1, 2025.

## Functional Requirements
The API must expose three new endpoints—`/intensity/live`, `/intensity/forecast`, and `/audit/trace`—with sub-500ms p99 latency across all regions. Data ingestion will pull from 42 Independent System Operators (ISOs) and the ENTSO-E Transparency Platform. Key capabilities include:

- Historical lookbacks of up to 90 days with hourly granularity
- Forecast windows extending 72 hours ahead using TerraGrid’s internal ML model, codenamed Orion
- Role-based access controls tied to SCIM provisioning for clients like Northvolt and RWE
- Emissions attribution using the marginal operating emissions rate (MOER) methodology adopted in January 2024

All responses must be ISO 8601 compliant and include provenance metadata linking back to source generators.

## Non-Functional & Security
The platform must maintain 99.99% uptime during the Q3 rollout and encrypt data at rest using AES-256. Penetration testing by external firm Korr Security is scheduled for August 15, 2025. Rate limits will start at 10,000 requests per minute per API key, scaling automatically via Kubernetes HPA clusters running in eu-west-1 and us-east-2.

## Success Metrics & Timeline
We will measure adoption through API call volume, targeting 5 million requests per day by November 2025. Customer satisfaction must remain above 4.5/5 in quarterly surveys. Engineering will freeze feature development on September 10, 2025, followed by a two-week beta with pilot partners. General availability is set for October 1, 2025.
