# CipherVault Security Changelog — Q1 2024 Release Notes

## Release Overview
CipherVault Security is publishing build 4.2.1 on March 14, 2024. This quarter's update focuses on hardening the Sentinel endpoint agent against emerging supply-chain exploits, expanding coverage for cloud-native workloads, and resolving regression defects reported by enterprise clients in the energy and financial sectors. All modules have passed internal red-team validation led by Dr. Elena Vasquez.

## Threat Intelligence Updates
The research team added 312 new behavioral signatures targeting ransomware affiliates observed since January. Key improvements include:

- Detection logic for the "Midnight Frost" loader variant first documented on February 2, 2024
- Heuristic coverage for CVE-2024-2199 affecting legacy VPN concentrators
- Enhanced YARA rules for malicious OAuth consent-grant applications in Microsoft 365
- Updated IP reputation lists containing 14,892 confirmed malicious nodes
- Correlation rules linking lateral movement to abnormal Kerberos AS-REQ patterns

These updates reduce mean-time-to-detect (MTTD) by an estimated 18 percent across monitored environments.

## Platform Improvements
Engineers rebuilt the policy engine to support granular micro-segmentation rules without requiring agent restarts. Memory footprint in the Linux daemon dropped by 22 MB under sustained load. The management console now renders real-time topology maps for networks exceeding 50,000 endpoints, and API rate limits were raised to 2,400 requests per minute for Enterprise tier subscriptions.

## Security Fixes and Hardening
This release patches three high-severity issues: a privilege escalation path in the Windows installer (CVSS 7.8), an information disclosure vulnerability in debug logging, and a race condition during certificate rotation. Cryptographic modules were upgraded to OpenSSL 3.2.1, and all build artifacts are now signed with the new CipherVault Code-Signing Key (CSK-2024-Q1).

## Known Issues and Mitigations
One intermittent crash affects the macOS agent version 4.2.0 when scanning compressed archives larger than 4 GB. A temporary workaround is available through support ticket #CV-8842. Full remediation is targeted for build 4.2.2, scheduled for release on April 4, 2024.
