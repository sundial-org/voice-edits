# TraceLoop CLI: Real-Time Error Context Design Doc

## Problem Statement
TraceLoop targets the 14-second average context-switching penalty engineers face when debugging distributed failures. By October 2024, our internal telemetry at NovaDev showed 68% of incidents required cross-referencing logs, metrics, and traces across three separate tools.

## Architecture Overview
The CLI uses a thin Rust binary (v0.9.2) that streams gRPC events from the TraceLoop agent into a local SQLite cache. Key design choices:

- Sub-200ms P99 latency for query responses via columnar indexing
- Offline-first mode with 72-hour retention windows
- Native LSP integration for VS Code and Neovim via the `traceloop-lsp` binary

## Data Flow and Storage
Incoming spans are batched every 500ms, compressed with Zstandard, then written to `~/.traceloop/cache/`. The agent, named `tld-agent-v2`, runs as a systemd user service and exposes port 8443. We enforce AES-256-GCM for data at rest and mandate mTLS 1.3 for agent-to-CLI communication.

## Security and Compliance
All user tokens expire after 90 days. The audit log, stored in `/var/log/traceloop/audit.jsonl`, records every query with a SHA-256 fingerprint. We completed SOC 2 Type I review on March 14, 2024, and are targeting Type II by Q1 2025.
