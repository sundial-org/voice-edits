# Project Prometheus — Autonomous Navigation Stack 2024

## Overview
Project Prometheus is the 2024 autonomy architecture developed by the Cornell University Autonomous Racing Team (CART) for the Indy Autonomous Challenge. Our objective is to achieve reliable high-speed navigation on the Indianapolis Motor Speedway using a custom ROS 2 middleware layer running on NVIDIA Orin AGX units. The stack integrates LiDAR-based localization, dynamic obstacle tracking, and model-predictive path planning into a single deterministic pipeline capable of 150 mph closed-loop operation.

## System Architecture
The perception module fuses 128-channel Ouster OS1 LiDAR returns with dual 12 MP Basler industrial cameras at 30 Hz. Localization leverages an extended Kalman filter against a pre-mapped point cloud collected on March 14, 2024. The planning layer executes a 50 ms receding-horizon optimizer that outputs steering and throttle commands directly to the dSPACE MicroAutoBox. Key design choices include:
- Redundant CAN-FD buses with 1 ms cycle times for actuator safety
- Thermal-managed GPU nodes operating below 75 °C during sustained 20-minute stints
- A fail-safe watchdog that triggers hydraulic brake engagement within 80 ms of a heartbeat loss

## Team and Milestones
Dr. Elena Vasquez serves as principal investigator, with lead software engineer Marcus Chen directing the ROS 2 integration. The mechanical integration was completed by junior members Aisha Patel and Tom Reed during the winter build session. We reached our first closed-loop milestone on April 3, 2024, and completed validation runs totaling 340 miles at the Goodyear Test Track before the June software freeze.

## Testing and Validation
All modules are validated in our Hardware-in-the-Loop simulator using a 1:1 vehicle dynamics model calibrated with data from the 2023 Dallara AV-21 chassis. We maintain a regression suite of 47 unit tests and 12 integration scenarios executed nightly on Jenkins. Field testing is scheduled for August 12–18, 2024, followed by the official competition event in September.

## Getting Started
Clone the repository and initialize the ROS 2 workspace with `colcon build`. Ensure firmware version 3.2.1 is flashed to the Orin modules before connecting to the vehicle bus. For questions, contact marcus.chen@cornell.edu or open an issue in the internal GitLab tracker.
