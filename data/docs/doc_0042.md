# Meridian Wallet — Real-Time Settlement Architecture

## Overview
Meridian Wallet is a mobile-first payments platform targeting small-business treasury operations. The design objective is to reduce settlement latency from T+2 to T+0 while maintaining PCI-DSS Level 1 compliance. Lead architect Sarah Chen proposed the current microservices topology in March 2024 after the Q1 outage revealed bottlenecks in our legacy monolith.

## Core Requirements
The system must process 12,000 transactions per second at peak with p99 latency under 180 ms. Regulatory filing with the OCC is scheduled for October 14, 2024, requiring full audit logging and immutable ledger replication across three AWS regions.

- Double-entry ledger engine with optimistic locking
- Real-time fraud scoring via Meridian Risk API v2.3
- End-to-end AES-256-GCM encryption for PII
- Automated reconciliation jobs running every 90 seconds
- Disaster recovery RPO of 15 minutes and RTO of 30 minutes

## Data Model & Storage
Transaction records live in Amazon Aurora PostgreSQL with read replicas in us-east-1 and eu-west-2. Immutable audit trails are written to Amazon QLDB for tamper-evident history. The settlement table uses a composite key of (merchant_id, settlement_batch_id) to avoid hot-partition issues at scale.

## Security & Compliance
All service-to-service traffic uses mTLS via Linkerd 2.14. Tokenization maps PANs to vault references stored in HashiCorp Vault Enterprise. Quarterly penetration tests are contracted through Mandiant, with the next engagement beginning August 5, 2024. Access policies enforce role-based controls tied to Active Directory groups.

## Deployment & Milestones
Infrastructure is managed via Terraform 1.7 and GitHub Actions. Staging parity tests run nightly; production releases follow a blue-green pattern with automated rollback on error-rate thresholds exceeding 0.05%. Beta launch with 500 pilot merchants is targeted for November 2024, followed by general availability in Q1 2025.
