# Meridian Commerce Platform Changelog — v2.4.0

## Release Overview
On March 14, 2024, we shipped v2.4.0 across all production clusters serving the North American and EU-West regions. This release focuses on checkout velocity, inventory accuracy, and compliance readiness for the new PCI-DSS 4.0 framework. Engineering lead Sarah Chen signed off after a 19-day QA cycle that included 400 automated regression tests.

## New Features
We rebuilt the promotional engine from the ground up. Merchants can now stack up to three concurrent discount rules without manual coupon codes, and the new "Bundle Builder" UI supports dynamic pricing for multi-item carts over $150.

- Real-time inventory syncing across Shopify, WooCommerce, and our native Meridian storefront
- One-click VAT invoice generation for B2B buyers in Germany, France, and the Netherlands
- Custom webhook endpoints for warehouse management systems using the 2024-03 API schema
- Dark-mode analytics dashboard with exportable CSV reports

## Performance Improvements
Average page-load time for product detail pages dropped from 2.8 seconds to 1.4 seconds after we migrated static assets to the CloudFront edge network. Database query latency during peak traffic—measured between 18:00 and 21:00 UTC—fell by 34 percent. Cart abandonment tracking now processes events within 50 milliseconds rather than polling every five minutes.

## Security and Compliance
All payment flows were upgraded to Stripe Connect 2024-01-15. We rotated every API key, enforced mandatory 2FA for admin roles above the "Editor" tier, and completed the external penetration audit conducted by RedShield Consulting on February 28, 2024. No critical vulnerabilities were found; two medium-severity issues in the coupon validation module were patched within 72 hours.

## Bug Fixes
We resolved the race condition that caused duplicate orders when customers clicked the buy button twice during high-latency sessions. Tax calculation for Alaska and Hawaii now rounds correctly to the nearest cent. Mobile Safari users will no longer see a broken sticky header on scroll, and the bulk-export tool supports files larger than 2 GB.

## Deprecation Notice
The legacy SOAP endpoint at `/v1/orders` will reach end-of-life on June 30, 2024. Migrate to the RESTful `/v2/orders` resource using the migration guide published in the developer portal. Support tickets referencing the deprecated protocol will receive automated guidance starting May 1, 2024.
