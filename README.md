# voice-edits

Fine-tuning `thinkingmachines/Inkling` (via [Tinker](https://tinker-console.thinkingmachines.ai))
to turn **spoken instructions** into **exact markdown document edits**.

Given a document and a voice memo ("rename the Future Work section to Roadmap
and delete the sentence about mobile support"), the model emits SEARCH/REPLACE
edit blocks that apply deterministically. See [RESULTS.md](RESULTS.md) for
numbers: 65.8% → **85.0%** exact-match after 45 LoRA steps, with **3.5× fewer
completion tokens**.

## Setup

Requires Python 3.11+, PyTorch ≥ 2.10, the `tinker` SDK, and the `tml-renderers`
wheel from the TML partner bundle. macOS is fine for everything (training runs
remotely; TTS uses macOS `say`).

```bash
python3.11 -m venv .venv && source .venv/bin/activate
pip install torch fastapi uvicorn pillow
pip install <tinker-sdk-wheel> <tml_renderers-wheel-for-your-platform>
cp .env.example .env   # fill in TINKER_API_KEY (+ OPENAI_API_KEY for neural TTS)
```

## Reproduce

```bash
python scripts/spike_audio.py         # sanity: say -> wav -> render -> sample
python scripts/gen_docs.py 300        # model generates its own doc corpus
python scripts/gen_edits.py 4         # 1200 verified (instruction, edit) pairs
python scripts/tts_batch.py           # macOS voices for train/val
python scripts/tts_openai.py          # OpenAI voices; test = held-out nova/shimmer
python scripts/run_eval.py --split test --mode voice          # zero-shot baseline
python scripts/train_sft.py --epochs 3                        # LoRA SFT (~25 min)
python scripts/run_eval.py --split test --mode voice \
  --model-path "$(tail -1 out/checkpoints.txt | cut -f2)"     # tuned eval
```

Design notes: ground-truth edits are generated programmatically (zero label
noise) and verified to apply; the test set uses TTS voices *and* instruction
phrasings never seen in training. `data/docs` and `data/examples.jsonl` are
checked in; wavs are regenerated by the TTS steps.
