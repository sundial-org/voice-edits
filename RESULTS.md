# Voice → document edits: fine-tuning results

Task: given a markdown document + a **spoken instruction (audio)**, emit exact
SEARCH/REPLACE edit blocks. Success = edits apply cleanly and the resulting
document matches the ground-truth target byte-for-byte (after whitespace
normalization).

## Setup

- Model: `thinkingmachines/Inkling` via Tinker, LoRA rank 32, lr 1e-4, 3 epochs,
  batch 64 (45 steps, ~25 min). Audio rendered client-side by `tml-renderers`.
  (Retrained after the launch-day rename from `thinkingmachines/Model`; numbers
  below are the Inkling run and match the preview run within ~1pt.)
- Data: fully synthetic, no customer data. 300 markdown docs generated by the
  model itself; 1,200 programmatic edit examples over 9 edit types
  (rename/delete/insert/move/replace/typo-fix; 25% multi-edit), each verified
  to apply cleanly. 960 train / 120 val / 120 test.
- Voice: train = 9 macOS voices + 4 OpenAI voices; **test = 2 held-out OpenAI
  voices (nova, shimmer) never seen in training**, plus disjoint instruction
  phrasings. Speed/rate varied throughout.

## Results (120-example held-out test set, temp 0)

| model | format-valid | applies cleanly | exact match | mean completion tokens |
|---|---|---|---|---|
| base (voice) | 113/120 | 113/120 | 79/120 (65.8%) | 214 |
| base (typed text) | 103/120 | 103/120 | 82/120 (68.3%) | 219 |
| SFT epoch 1 | 120/120 | 116/120 | 98/120 (81.7%) | 62 |
| **SFT epoch 2** | **120/120** | **117/120** | **102/120 (85.0%)** | **62** |

- Error rate cut by more than half (34.2% → 15.0%); format compliance perfect
  after SFT.
- **3.5× fewer completion tokens** — the tuned model answers directly in-format.
- Zero-shot, spoken instructions are near text parity (65.8% vs 68.3%): the
  base model's audio understanding costs only ~2.5pt vs typed input.
- Remaining failures skew to semantically fuzzy ops (delete_sentence,
  rename_entity) and are balanced across both held-out voices.

## Repro

`scripts/`: gen_docs → gen_edits → tts_batch + tts_openai → run_eval (baseline)
→ train_sft → run_eval --model-path. Eval JSONs in `out/` (`ink_*.json`);
best checkpoint: `out/checkpoints.txt` epoch2.
